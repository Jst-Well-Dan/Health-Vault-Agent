# visits 写入指南

本文给后续 AI agent 使用，目标是把一次就诊、复查、治疗或检查稳定写入 `visits` 表，并为前端提供可直接展示的结构化字段。不要依赖前端正则或页面逻辑从诊断文本中推断严重程度。

## 表结构语义

`visits` 当前字段：

| 字段 | 写入要求 |
| --- | --- |
| `member_key` | 成员 key，必须来自已存在成员。 |
| `date` | 就诊、检查或治疗日期，格式 `YYYY-MM-DD`。同一报告有多个日期时，优先用实际就诊/采样日期。 |
| `hospital` | 医院、诊所或宠物医院名称。没有明确机构时留空，不要猜。 |
| `department` | 科室、门诊、检查部门或宠物医疗项目。没有明确科室时留空。 |
| `doctor` | 医生姓名。没有明确医生时留空。 |
| `chief_complaint` | 本次记录标题。优先写主诉；没有主诉时写报告目的，如 `血常规复查`、`免疫抗体及术前检查`。 |
| `severity` | 本次治疗/就诊严重程度，只能写 `严重` / `轻微` / `一般` / `NULL`。 |
| `diagnosis` | JSON 字符串数组。写医生诊断、明确病名或报告结论；没有诊断时写空数组。 |
| `notes` | 一句话摘要，写最关键的医嘱、检查结论、异常结果或后续安排。 |
| `source_file` | 来源 MD 或 PDF 文件名，便于追溯。 |
| `note_full` | 精细提取后的医生诊断、诊疗意见、治疗方案。当前字段由后续同步任务维护，若脚本支持可一并写入。 |

## 核心原则

1. `visits` 表描述“一次医疗事件”，不要把同一报告中的每个指标拆成多条 `visits`。
2. 检验指标写入 `lab_results`，处方/用药写入 `meds`，附件写入 `attachments`；`visits.notes` 只放摘要。
3. `severity` 是结构化判断结果，必须写入数据库。前端只负责显示，不负责推断。
4. 严重程度不是永久病情分级，而是“本次就诊/治疗记录需要被关注的程度”。
5. 不要仅凭疾病名称定级。同一个诊断在复诊稳定期和急性未控制期可以不同。
6. 如果报告信息不足，`severity` 可以留空，但不要用含糊词替代枚举值。
7. 宠物记录和人类记录使用同一套 `severity` 枚举，但判断时必须参考物种、报告参考范围和医生结论。

## severity 判断口径

### 严重

满足以下任一条件时，通常写 `严重`：

| 依据 | 示例 |
| --- | --- |
| 医生明确写出未控制、强阳性、重度、疑似重大疾病、明显炎症、急性加重。 | `支气管哮喘(未控制)`、`疑似猫传腹`、`猫皮屑重度过敏` |
| 关键指标显著异常，且与本次诊断或治疗直接相关。 | FeNO 明显升高、SAA 极高、肺功能激发试验强阳性 |
| 需要启动系统性治疗、密切复查、专科治疗或对生活/安全有明显影响。 | 启动脱敏治疗、规律控制哮喘用药、疑似 FIP 评估 |

不要因为普通诊断名称本身就写 `严重`。必须有报告结论、异常指标或治疗强度支撑。

### 轻微

满足以下情况时，通常写 `轻微`：

| 依据 | 示例 |
| --- | --- |
| 有症状或异常，但没有急危重证据。 | 膝关节疼痛、轻度 ALT 升高、SAA 恢复期仍略高 |
| 需要短期用药、观察、复查或生活方式处理。 | 外用/口服止痛药 7 天、建议必要时核磁、术前指标关注 |
| 慢病或炎症处于恢复/随访阶段，但仍存在异常。 | 猫传腹恢复期复查，指标明显好转但未完全正常 |

`轻微` 不是“不重要”，而是提示需要关注但优先级低于 `严重`。

### 一般

满足以下情况时，通常写 `一般`：

| 依据 | 示例 |
| --- | --- |
| 常规复诊、维护治疗、预防性检查或状态稳定。 | 脱敏治疗按计划注射并留观 |
| 报告没有明显异常，或异常与本次处置无强相关。 | 常规免疫/体检记录，医生未提示特殊风险 |
| 主要作用是归档和连续记录。 | 周期性治疗记录、常规筛查 |

`一般` 可用于绿色徽章，表示本次记录为常规/稳定状态，不表示疾病已经治愈。

## 字段写入规则

### chief_complaint

优先级：

1. 报告原文主诉。
2. 医生诊断或就诊目的的短标题。
3. 检查报告目的。
4. 文件名清洗后的标题。

示例：

| 原始信息 | `chief_complaint` |
| --- | --- |
| 主诉：哮喘复查 | `哮喘复查` |
| 脱敏治疗单 | `变应原特异性免疫治疗` |
| 宠物 SAA、血常规复查 | `血常规复查` |
| 免疫抗体、术前生化、血常规 | `免疫抗体及术前检查` |

### diagnosis

写医生明确诊断或报告明确结论：

- 保持为数组，即使只有一个诊断。
- 不把症状、用药、检查项目写成诊断。
- 没有诊断时写 `[]`，不要写 `["未见诊断"]`。
- 诊断名称尽量使用报告原文，必要时清理多余空格和 OCR 符号。

### notes

`notes` 应该是一句话摘要，便于列表页阅读：

- 写关键异常：`FeNO 185.4 ppb`、`SAA 169.44 mg/L`。
- 写关键医嘱：`建议规律用药并复查`、`必要时核磁检查`。
- 不粘贴整段报告。
- 不把所有 lab 指标堆进 `notes`。

### source_file

- 优先写原始 PDF 文件名。
- 若导入来源是 MD，也可以写 MD 文件名。
- 文件名必须能追溯到 `data/reports/...` 下的真实文件。

```json
{
  "visit": {
    "member_key": "chunzi",
    "date": "2025-09-16",
    "hospital": "首都医科大学附属北京安贞医院通州院区",
    "department": "呼吸与危重症医学科门诊",
    "doctor": "李阁",
    "chief_complaint": "哮喘复查",
    "severity": "严重",
    "diagnosis": ["支气管哮喘(未控制)", "过敏性鼻炎"],
    "notes": "肺功能激发试验强阳性；FeNO 185.4 ppb，建议规律用药并复查。",
    "source_file": "20250902_安贞医院_春子_哮喘.pdf"
  },
  "labs": [],
  "meds": [],
  "attachments": []

```

## 复核清单

写入或更新 `visits` 前，逐项检查：

- `member_key` 是否存在。
- `date` 是否为实际就诊/检查日期。
- `chief_complaint` 是否足够短且能代表本次记录。
- `severity` 是否只使用 `严重`、`轻微`、`一般` 或留空。
- `diagnosis` 是否为数组，且只包含明确诊断/结论。
- `notes` 是否是一句话摘要，不替代 `lab_results` 或 `meds`。
- `source_file` 是否能追溯到真实报告。
- 本次记录相关指标、用药和附件是否分别写入对应表。
